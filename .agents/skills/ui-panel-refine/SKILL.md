---
name: ui-panel-refine
description: 当任务是优化右侧属性面板、左侧列表面板、输入框排版、标签对齐、空间压缩、滚动区布局时使用。适用于局部 UI 整理和最小范围样式调整。不适用于新增大功能、全局重构或改动整个设计系统。
---

目标：
在不扩大改动范围的前提下，优化编辑器面板布局与可读性。

适用场景：

- 右侧属性面板太丑、太松散、浪费空间
- 标签与输入框未对齐
- 多个字段可以并排但当前分两行
- 左右栏宽度需要收紧
- 搜索框和筛选器需要更紧凑地排布
- 需要保留现有功能，只做布局和交互微调

不适用场景：

- 新增新的对象类型
- 大范围重构状态管理
- 重写整个 UI 组件体系
- 新增复杂业务逻辑

执行步骤：

1. 先定位和当前面板直接相关的组件、样式和渲染入口。
2. 只阅读和当前布局问题直接相关的代码，避免扩散到无关文件。
3. 先总结当前问题，再提出最小改动方案。
4. 优先通过局部布局调整解决问题：
   - 减少间距
   - 合并同类字段到一行
   - 调整 label 宽度与输入框高度
   - 优化 section 内边距和滚动区
5. 保留现有交互能力，不要删掉已有可用功能。
6. 修改完成后，检查是否引入新的类型错误或明显的 UI 回退。
7. 输出时说明：
   - 改了哪些文件
   - 每个文件改了什么
   - 还存在哪些可继续优化的点

完成标准：

- 面板更紧凑
- 关键信息更对齐
- 不破坏现有功能
- 改动范围可解释、可回退
