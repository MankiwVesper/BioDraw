---
name: spec-sync
description: 当任务是根据当前代码状态更新项目 spec、设计说明、开发说明或模块文档时使用。适用于已有现成代码实现，需要同步文档内容、补充结构说明、更新字段或交互描述的场景。不适用于从零编写全新方案，或在没有查看代码的情况下猜测实现细节。
---

目标：
根据当前代码状态，更新对应的 spec 或说明文档，确保描述与实现一致。

执行原则：

1. 先阅读与当前功能直接相关的代码文件。
2. 不根据旧 spec 盲目续写，必须以当前代码为准。
3. 只更新与当前任务相关的章节，不顺手重写整份文档。
4. 明确说明：
   - 哪些内容是当前代码已实现的
   - 哪些内容仍是待实现或建议项
5. 输出时给出：
   - 更新了哪些章节
   - 每一处更新的依据是什么
   - 是否还有需要补充确认的点

完成标准：

- 文档内容与当前代码一致
- 不混入臆测实现
- 更新范围清晰
- 便于后续继续开发
